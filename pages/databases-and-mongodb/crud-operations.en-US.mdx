CRUD Operations with MongoDB:
1. Creating (Inserting) Data:
javascript
Copy code
// Insert a document into a collection
db.users.insertOne({
  name: "John Doe",
  age: 30,
  email: "john.doe@example.com"
});
2. Reading (Querying) Data:
javascript
Copy code
// Find all documents in the users collection
db.users.find();
3. Updating Data:
javascript
Copy code
// Update a document in the users collection
db.users.updateOne(
  { name: "John Doe" },
  { $set: { age: 31 } }
);
4. Deleting Data:
javascript
Copy code
// Delete a document from the users collection
db.users.deleteOne({ name: "John Doe" });
CRUD Operations with MySQL:
1. Creating (Inserting) Data:
sql
Copy code
-- Insert a record into the users table
INSERT INTO users (name, age, email) VALUES ('John Doe', 30, 'john.doe@example.com');
2. Reading (Querying) Data:
sql
Copy code
-- Select all records from the users table
SELECT * FROM users;
3. Updating Data:
sql
Copy code
-- Update the age of a user in the users table
UPDATE users SET age = 31 WHERE name = 'John Doe';
4. Deleting Data:
sql
Copy code
-- Delete a user from the users table
DELETE FROM users WHERE name = 'John Doe';
These examples illustrate the basic syntax for CRUD operations in both MongoDB and MySQL. The exact commands might vary depending on the specifics of your database schema, but the core concepts of create, read, update, and delete remain consistent across databases. Always remember to handle errors appropriately, and ensure that your application has the necessary permissions for performing these operations on the database.
